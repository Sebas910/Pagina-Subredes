⚡ MÓDULO 0 · EL SECRETO DEL SISTEMA BINARIO

Toda la magia del subnetting se reduce a una sola base: el número 2. Las IPs IPv4 son 32 bits (unos y ceros). Si dominas las potencias de 2, nunca tendrás que memorizar tablas complejas.

📜 FÓRMULAS DE ORO
SUBREDES
2n

n = bits "prestados" de la porción host.

HOSTS USABLES
2h − 2

h = bits restantes. −2 por IP de Red y Broadcast.

🎮 SIMULADOR · POTENCIAS DE 2
BITS ASIGNADOS 4
VALOR BRUTO (2ⁿ) 16
HOSTS USABLES 14
TABLA RÁPIDA
2¹=2 2²=4 2³=8 2⁴=16 2⁵=32 2⁶=64 2⁷=128 2⁸=256 2⁹=512 2¹⁰=1024
🦘 EL TRUCO DEL SALTO DE RED

Para saber de cuánto en cuánto "saltan" las subredes, usa esta fórmula rápida:

256 − M

Donde M = valor del octeto modificado en la máscara.

EJEMPLO

Máscara /26 → último octeto = 192

Salto = 256 − 192 = 64

→ Redes avanzan de 64 en 64

📦 MISIÓN 1 · FLSM — TAMAÑO FIJO

Todas las subredes se cortan del mismo tamaño. Simple, pero puede desperdiciar IPs.

▶ PLANTEAMIENTO

Dirección base: 192.168.1.0/24
Dividir en exactamente 4 subredes

01
ENCONTRAR LOS BITS (n)

¿Qué valor de 2n es ≥ 4 redes?

✓ RESULTADO

21=2 (insuficiente)  ·  22=4 ✓
Necesitamos n = 2 bits

02
FORJAR LA NUEVA MÁSCARA

/24 original + 2 bits prestados = ?

✓ RESULTADO

/24 + 2 = /26
Último octeto: 128 + 64 = 192
Máscara decimal: 255.255.255.192

03
CALCULAR EL SALTO DE RED

256 − (octeto modificado de máscara)

✓ RESULTADO

256 − 192 = 64
Las subredes avanzan de 64 en 64 en el último octeto.

04
CONSTRUIR LA TABLA FINAL

ID Red → +1 primera IP → +salto−2 última IP → +salto−1 broadcast

✂️ MISIÓN 2 · VLSM — A LA MEDIDA

Cada subred se ajusta exactamente a lo que se necesita. Cero desperdicio de IPs.

REGLA DE ORO — ORDENAR PRIMERO

En VLSM es OBLIGATORIO ordenar de MAYOR a MENOR antes de calcular. Si calculas en orden arbitrario, las IPs se solaparán y el ejercicio estará incorrecto.

✓ ORDEN ESTABLECIDO
1000
→
500
→
2

Calculamos en este orden exacto, uno tras otro.

02
RED MÁS GRANDE · 1000 HOSTS

Iniciamos en: 172.16.0.0

✓ PASO A PASO
🔍 Bits (h): 2h − 2 ≥ 1000. 210=1024 ✓ → h = 10 bits
🛡 Máscara: 32 − 10 = /22 → 255.255.252.0
🦘 Salto: 256 − 252 = 4 (en el 3er octeto)
🏆 RESULTADO RED 1000

ID Red: 172.16.0.0/22
Broadcast: 172.16.3.255
→ Siguiente red empieza en: 172.16.4.0

03
RED MEDIANA · 500 HOSTS

Continuamos en: 172.16.4.0

✓ PASO A PASO
🔍 Bits (h): 2h − 2 ≥ 500. 29=512 ✓ → h = 9 bits
🛡 Máscara: 32 − 9 = /23 → 255.255.254.0
🦘 Salto: 256 − 254 = 2 (en el 3er octeto)
🏆 RESULTADO RED 500

ID Red: 172.16.4.0/23
Broadcast: 172.16.5.255
→ Siguiente red empieza en: 172.16.6.0

04
RED PEQUEÑA · 2 HOSTS (ENLACE WAN)

Continuamos en: 172.16.6.0

✓ PASO A PASO
🔍 Bits (h): 2h − 2 ≥ 2. 22=4 ✓ → h = 2 bits
🛡 Máscara: 32 − 2 = /30 → 255.255.255.252
🦘 Salto: 256 − 252 = 4 (en el 4to octeto)
🏆 RESULTADO RED 2

ID Red: 172.16.6.0/30
Broadcast: 172.16.6.3
→ IPs usables: .1 y .2 — exactamente 2, sin desperdicio!

4 Puntos Importantes a tomar en cuenta.

1
La Gran Diferencia

FLSM Máscara fija. Corta pedazos iguales. Puede desperdiciar IPs.

VLSM Máscara variable. Corta a la medida exacta. Optimiza el espacio al 100%.

2
Hosts Usables
2h − 2

Restamos 2: IP de Red y Broadcast no son asignables.

3
El Truco del Salto
256 − M

256 menos el octeto modificado de la máscara = salto entre redes consecutivas.

4
Mandamiento VLSM

"Antes de calcular,
ordenar de MAYOR a MENOR."
